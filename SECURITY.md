# Security Policy

## Supported Scope

This repository contains the public reference implementation only. Do not open public issues with:

- live access data
- GitHub tokens
- service account credentials
- internal project identifiers
- screenshots containing real repository, team, or user names

## Reporting A Vulnerability

Report security issues privately to the maintainer before public disclosure.

Minimum report content:

- affected file or component
- impact summary
- reproduction steps
- suggested fix, if available

Do not include working credentials in the report.
